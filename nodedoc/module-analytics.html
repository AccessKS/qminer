Example
// import module
var analytics = require('qminer').analytics;
// REGRESSION WITH SVR
// Set up fake train and test data.
// Four training examples with, number of features = 2
var featureMatrix = new la.Matrix({rows:2, cols:4});
// Regression targets for four examples
var targets = new la.Vector({vals:4});
// Set up the regression model
var SVR = new analytics.SVR({verbose:true});
// Train regression
SVR.fit(featureMatrix, targets);
// Save the model to disk
SVR.save('svr.bin');*
// Set up a fake test vector
var test = new la.Vector({vals:2});
// Predict the target value
var prediction = SVR.predict(test);

Type:

Properties:
Name Type Attributes Default Description
svcParam.c number <optional>
 1.0 Cost parameter. Increasing the parameter forces the model to fit the training data more accurately (setting it too large may lead to overfitting)
svcParam.j number <optional>
 1.0 Unbalance parameter. Increasing it gives more weight to the positive examples (getting a better fit on the positive training examples gets a higher priority). Setting c=n is like adding n-1 copies of the positive training examples to the data set.
svcParam.batchSize number <optional>
 10000 Number of examples used in the subgradient estimation. Higher number of samples slows down the algorithm, but makes the local steps more accurate
svcParam.maxIterations number <optional>
 10000 Maximum number of iterations
svcParam.maxTime number <optional>
 1.0 Maximum runtime in seconds
svcParam.minDiff number <optional>
 1e-6 Stopping criterion tolerance
svcParam.verbose boolean <optional>
 false Toggle verbose output in the console
